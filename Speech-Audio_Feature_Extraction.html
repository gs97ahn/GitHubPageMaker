Speech - Audio Feature Extraction

Audio Feature Extraction

Feature Extraction

  • What is feature extraction?
    • tall and big. 4 feet. 5 horns -> giraffe
    • long nose -> elephant
  • Reason for feature extraction
    • decrease in data size
    • increase in recognition

Audio Feature Extraction

  • Why wav format files are not directly inputted
    • contains noise
    • too much data size

Audio Feature Commonly Used

  • Time domain feature
    • Zero crossing
    • Tempo
  • Frequency domain features
    • Spectral Centroid
    • Spectral Spread
    • Spectral Flatness
    • Spectral Envelope
  • Time-frequency feature
    • Spectrogram(STFT)
    • Log Mel-spectrogram
    • MFCC
    • Constant-Q transform(CQT)
    • Chroma

Caution with Feature

  • Time-frequency expression –> STFT (Short Time Fourier Transform)
  • Frequency expression similar to human ear –> Mel scale
  • Sound pressure expression similar to human ear –> Log scale
  • STFT + Mel Scale + Log Scale –> Log Mel spectrogram

STFT (Short Time Fourier Transform)

  1. Divide by section
  2. Apply Window
  3. FFT
  4. Graph

Librosa

Librosa

  • python package for music and audio analysis

Spectrogram

# spectrogram
stft = librosa.stft(audio, n_fft=n_fft, hop_length=hop_length)
sp = np.abs(stft)**2
sp_db = librosa.power_to_db(sp, ref=np.max, top_db=150)

Spectrogram vs. Spectrogram(dB)

  • Make sure to use dB because the model will be trained more accurately

Mel spectrogram

Mel-scale

  • Audio level that is heard by human ears (Mel -> Melody)
  • 1000 mels = 1000 Hz tone

Example

  • 400.00 Hz -> 509.38 Mel 10 Hz -> 10.2 Mel

  • 410.00 Hz -> 519.58 Mel

  • 2000.00 Hz -> 1521.36 Mel 10 Hz -> 4.17 Mel
  • 2010.00 Hz -> 1525.53 Mel

librosa.feature.melspectrogram

# melspectrogram
melsp = librosa.feature.melspectrogram(y = audio, sr = fs, n_fft = n_fft, hop_length = hop_length, n_mels = 80)

librosa.power_to_db

# log melspectrogram
logmelsp = librosa.power_to_db(melsp, ref=np.max, top_db=150)

MFCC

MFCC

  • Normally used for audio feature expression
  • Mel-Frequency Cepts(Spec)tral Coefficients

Naming

  • Spectrum -> Cepstrum
  • Frequency -> Quefrency
  • Filtering -> Liftering
  • Harmonic -> Rhamonic

MFCC Extraction Process

  • Mel spectrum -> Log -> Inverse FT(DCT: Discrete Cosine Transform)

librosa.feature.mfcc

# mfcc
mfccs = librosa.feature.mfcc(y=audio, sr=fs, n_mfcc=40)

Feature Comparison

Case 1

  • Combination
    • Dataset: ESC-10, ESC-50, UrbanSound8K
    • Model: AlexNet, GoogLeNet
    • Sampling rate: 8k, 16k, 32k
    • Features: Mel spectrogram, MFCC
    • Frame length: 20, 30, 40, 50ms

Sampling rate

  • 16 kHz is best for AlexNet and GoogLeNet

Frame length

  • 30 ms is best for AlexNet and GoogLeNet

Case 2

  • Features
    • STFT
    • Mel-STFT
    • CQT(Constant-Q Transform)
    • CWT(Continuous Wavelet transform)
    • MFC

ESC-50 Recognition Rate Comparison

  • Mel-STFT is best

UrbanSound8K Recognition Rate Comparision

  • Mel-STFT is best

Best Feature?

Optimized Feature

  • Mel spectroam and MFCC are modeled as it is similar to how huamns perceive audio
  • There is a need to construct a new feature specific for mic and algorithm

What is Optimized Feature for Machine Learning

  • Feature does not need to immitate how humans perceive information
  • Research for feature specificially for machine learning is neccessary
