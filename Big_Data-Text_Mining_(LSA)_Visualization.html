Big Data - Text Mining (LSA) Visualization

VS model in practice

  • Document & query are represented by term vectors
    • Terms are not necessarily orthogonal to each other
      • Synonymy: car vs. automobile
      • Polysemy: fly (action vs. insects)

Choosing basis for VS model

  • A concept space is preferred
    • Semantic gap will be bridged

How to build such as space

  • Automatic term expansion
    • Construction of thesaurus
      • WordNet
    • Clustering of words
  • Word sense disambiguation
    • Dictionary-based
      • Relation between a pair of words should be similar as in text and dictionary’s description
    • Explore word usage context
  • Latent Semantic Analysis
    • Assumption: there is some underlying latent semantic structure in the data that is partially obscured by the randomness
    • It means: the observed term-document association data is contaminated by random noise

Solution

  • Low rank matrix approximation

Latent Semantic Analysis (LSA)

  • Low rank approximation of term-document matrix CMxN
    • Goal: remove noise in the observed term-document association data
    • Solution: find a matrix with rank k which is closest to the original matrix in terms of Frobenius norm

Basic concepts in linear algebra

  • Symmetric matrix
    • C = CT
  • Rank of a matrix
    • The number of linearly independent rows (columns) in a matrix CMxN
    • rank(CMxN) <= min (M,N)
  • Eigen system
    • For a square matrix CMxM
    • If Cx = λx, x is called the right eigenvector of C and λ is the corresponding eigenvalue
  • For a symmetric full-rank matrix CMxM
    • We have its eigen-decomposition as
      • C = QΛQT</sub>
      • where the columns of Q are the orthogonal and normalized eigenvectors of C are Λ is a diagonal matrix whose entries are the eigenvalues of C
  • Singular value decomposition (SVD)
    • For matrix CMxN with rank r, we have
      • C = U∑VT
      • where UMxr and VNxr are orthogonal matrices, and ∑ is a r x r diagonal matrix, with ∑ii = √(λi) and λ1 … λr are the eigenvalues of CCT
    • We define CkMxN = UMxk∑kxkVTNxk
      • where we place ∑ii in a descending order and set ∑ii = ∑ii = √λi for i <= k, and ∑ii = 0 for i > k

Latent Semantic Analysis (LSA)

  • Solve LSA by SVD

  • Procedure of LSA
    1. Perform SVD on document-term adjacency matrix
    2. Construct CkMxN by only keeping the largest k singular values in ∑ non-zero

What are those dimensions in LSA

  • Principal component analysis (PCA)

LSA beyond text

  • Collaborative filtering
    • User item matrix stores for each user the rating for the item

Data Visualization: Fundamentals for visualization

Overview

  • Definition: To represent the data (or result) for better understanding
    • How can I represent what my data (or results) shows?
  • Why visualize?
    • Giving better, clear understanding to audiences
    • Can get perceptual motivations
    • …

Principles on Visualization

  • Be true to your research
    • Design your display to illustrate a particular point
  • Maximize information, minimize ink
    • Use the simplest possible representation for the bits you want to convey
  • Organize hierarchically
    • What should a viewer see first? what if they look deeper?

Data Types & Aesthetics

  • Data Types
    • Norminal - values from an unordered set, e.g., color, profession
    • Ordinal - values from an ordered set, e.g., military or academic rank
    • Numeric - real numbers, e.g., integer or real numbers
  • Aesthetics
    • Shape
    • Color
    • Size
    • Position (Axis)

  • Visualization: Connect data type(s) into Aesthetics

How to Choose Colors?

  • “Qualitative”, “Sequential”, “Diverging” colors

Note ❗️

Good Example

Position (Axis)

  • Cartesian Coordinate system
    • Linear vs. Log-scale

  • Polar Coordinate system
    • Good to see changes of values for continuous data

Data Visualization: Basic plots

Popular Plots for Visualization

Histograms

  • One dimensional
  • Shows shape by binning a continuous distribution
  • Grouped histogram can indicate multiple discrete categories -> Good for comparison

Pie Chart

  • A whole split into parts
  • Emphasizes that all parts sum to a constant
  • Single dimension with discrete categories

Stacked Bar Plot

  • Wholes split into parts
  • Easy to compare
    • Often better than pie chart
  • Can have multiple discrete dimensions

Venn diagram

  • Shows overlap between discrete groups
  • Sometimes the only way to display overlapping sets
    • No overlapping ratio

Scatter Plots

  • Relationship between observations on two continuous dimensions
  • Can show multiple groups
  • Can show trend lines etc.
  • Uninformative with too much data

Line Plot

  • Good for showing one variable (e.g., time) as continuous even though you have discrete measures.
  • Can compare several discrete groups

Bar plots

  • Can be used for lots of discrete grouping factors
  • Natural semantics of grouping
  • Conceals data distribution (i.e., Less Informative)

(CF) Issues on bar chart

  • X-axis vs. legend: please think what to compare first
  • Usually, too much is bad
    • Separation may be better

Heatmap

  • Works very well when there are natural semantics or comparing intensity of data
  • Color mapping can be problematic
    • Usually, sequential or diverging colors are good

Bubble Plot

  • Extension of scatter plot (+ one dimension)
    • Size is not perfectly quantitative

Data Visualization: Visualization for Distribution

Visualizing Distributions

  • The first step to observe “how data is shaped” without any manipulation
    • Power-law, normal, …
  • How?
    • Using error bars
    • Histograms after binning -> kernel density estimation
    • Box, violin, strip, sina, ridgeline
    • CDF & QQ-plots

Using Error Bars for Distribution

  • Mean + standard deviation
  • Can be used for lots of discrete grouping factors
    • Good to compare
  • Conceals data distribution (i.e., Less Informative)

Histogram for Distribution

  • Binning is essential process
  • Good to see distribution of one variable
  • Different binning may result in different messages

Histogram to Kernel Density Estimation

  • Discrete variable -> Continuous trends

Comparing Distributions

How about 3 or more distributions?

  • Error bar is too lossy
  • Boxplot can be an answer

Violin plot is a super set of boxplot

Strip or Sina plot May Be Better

  • Strip: Using jitter to represent density
  • Sina: Violin + strip

If You Have More Distributions

  • Ridgeline plots can be a solution

Warning ❗️

You should have enough number of data

For raw (empirical) data, CDF is usually best

Empirical data with Theoretical fits

  • Quantile-Quantile (QQ) plots

Reference

  • Mining of Massive Datasets by Jure Leskovec, Anand Rajaraman, Jeff Ullman
