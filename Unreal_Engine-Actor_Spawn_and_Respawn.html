Unreal Engine - Custom Event and Event Dispatcher

Spawn and Respawn

  • Dynamically spawn an object
    • Once located in a map, it is static, meaning it does not move
    • Object that dynamically appear and disappear
      • Item that is collected should disappear
      • Cannon should disappear after it is launched from a cannon shooter
      • Enemy should constantly spawn
      • Player should respawn after death
  • Spawn and Respawn
    • Spawn: Object such as character or item that spawns dynamically
    • Respawn: Character that respawns after death

Actor Spawning

  • Location fixed with a constant is not flexible
  • Location information must have an actor
  • Location can be adjusted using a mouse
  • A typical actor: “Target Point” actor, “Player starter” actor, …

Implementation

  • Actor spawning
    • SpawnActor from Class
  • Transform
    • transform matrix
    • Location + Rotation + Scale
  • Extract actor’s transform
    • GetActorTransform

Destroy

  • Unnecessary actors
    • Resources(CPU, Memory) must not be wasted
  • Way to delete an actor
    • Destroy Actor
    • Using lifespan property
    • Using kill Z property

Using Timer

  • Using timer
    • After the specified time, the subroutine is called
  • Two types of timer function
    • Utilities > Timer > ‘Set Timer by Function Name’
    • Utilities > Timer > ‘Set Timer by Event’ (Recommended)
  • Timer function input
    • Function Name: Function / Custom event name string
    • Event = Connect delegate pin
    • Time: Wait time in seconds
    • Looping: Called every specified time
      • you can use other actions such as ResetTimer, UnsetTimer
    • Return Value: Timer handle returns
  • Other timer related function
    • Clear and Invalidate Timer by Handle: Deletes an infinite loop timer
    • Get Timer Elapsed(Remaining) Time by Handle: Returns remaining time
    • Paused(Unpaused) Timer by Handle: Pauses a timer

Using Hit Event

  • If the collision occurs in block form, not overlap, the default event does not occur
  • When block form of collision event occurs:
    • Check ‘Simulation Generates Hit Events’ in the component’s detail tab
  • Callback function
    • Add the ‘OnComponentHit’ to the specific component to event node in the node graph
  • Callback function in actor metrics
    • Add the ‘OnActorHit’ to the specific instance to event node in the node graph

Pawn and Controller

  • Difference between Pawn and Controller
    • Pawn: An existence, such as player or AI, in the world that has physical shape
    • Controller: Controller class that manages a pawn
      • Human Player or Computer Player(AI)
    • When a game starts, the specified pawn and controller is auto-generated
      • Controller is not visible but it is auto-generated(you can see it in the outliner)
  • Input Event Process
    • General way: Controller gets the input and applies to the pawn
    • Simple implementation way: Pawn itself gets the input and controls itself

Pawn’s Response

Pawn’s response

After pawn is destroyed, the pawn must respawn in the specified place to replay. However, the pawn’s controller does not disappear. Therefore, when the pawn respawn, the original controller must be reconnected.

Pawn and Controller Connection

  • Pawn and Controller should have one to one connection
  • In order for the controller to Possess a different pawn, it must UnPossess first

Event binding

  • Respawning after a player disappear
    • Define a custom event so the player’s pawn will respawn
    • Make it so the event is called when the pawn is destroyed
  • Event binding
    • Event bind OnDestroyed to the respawn event
    • However, pawn level is not placed, it is created upon it runs
