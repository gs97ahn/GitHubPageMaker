Computer Network - What is Inside a Router

The Four router components can be identified:

  • Input ports. An input port performs several key functions.
    • The physical layer function of terminating an incoming physical link at a router.
    • Link-layer functions needed to interoperate with the link layer at the other side of the incoming link.
    • Lookup functions at the input port
  • Output ports. An output port stores packets received from the switching fabric and transmits these packets on outgoing link by performing the necessary link-layer and physical-layer functions.

  • Routing processor. The routing processor performs control-plane functions.

  • Control plane
    • Router’s control functions:
      • Execute the routing protocols.
      • Respond to attached links that go up or down.
      • Communicate with the remote controller.
    • Management functions:
      • Operate at the millisecond or second timescale.

Information required for packet forwarding process:

  • Destination-based forwarding
  • Generalized forwarding

“Lookup” is arguably the most important action in input port processing, but many other actions must be taken as well:

  1. Physical- and link-layer processing must occur.
  2. The packet’s version number, checksum and tim-to-live field must be checked and the latter two fields rewritten.
  3. Counters used for network management must be updated.

2️⃣ Switching

Switching can be accomplished in a number of ways:

  • Switching via memory.
  • Switching via a bus.

  • Switching via an interconnection network.

  • Non-blocking: A packet being forwarded to an output port will not be blocked from reaching that output port.

3️⃣ Output Port Processing

4️⃣ Where Does Queuing Occur?

  • Packet loss: Occur when no memory is available to store arriving packets.

Input Queuing

  • Head-of-the-line (HOL) blocking: A queued packet in an input queue must wait for transfer through the fabric because it is blocked by another packet at the head of the line.

Output Queuing

  • Drop-tail: Drop of the arriving packet when there is not enough memory to buffer an incoming packet.
  • Active Queue Management (AQM): A number of proactive packet-dropping and -marking policies.
  • Random Early Detection (RED): One of the most widely studied and implemented AQM algorithm.
  • Packet scheduler: Choose one packet among those queued for transmission at the output port.

5️⃣ Packet Scheduling

First-In-First-Out (FIFO)

Priority Queuing

  • Non-preemptive priority queuing: A discipline that forces the transmission of a packet to be not interrupted once it has begun.

Round Robin and Weighted Fair Queuing (WFQ)

  • Work-conserving queuing: A discipline that forces the to never allow the link to remain idle whenever there are packets queued for transmission.
  • Weighted fair queuing (WFQ) discipline: Arriving packets are classified and queued in the appropriate per-class waiting area.
