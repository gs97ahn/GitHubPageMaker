Tag, Property and Property Value

  • HTML?
    • document starting with < html > < /html >
    • Composed of tag and property
  • Tag
    • < ul > < /ul > < li > < /li > < div > < /div > < a > < /a >, things that are wrapped with < >
  • Property and Property Value
    • < ul class=”class1” >
      • ul: tag, class: property, “class1”: property value

Get data using class property

from bs4 import BeautifulSoup

html_str = """
<html>
  <body>
    <ul class="class1">
        <li>line1</li>
        <li>line2</li>
        <li>line3</li>
    </ul>
    <ul class="class2">
        <li>line4</li>
        <li>line5</li>
        <li>line6</li>
    </ul>
  </body>
</html>
"""

soup = BeautifulSoup(html_str, "html.parser")

ul = soup.find("ul", {"class": "class2"})
print(ul)

Difference between class and id

  • id: unique in a web page
  • class: able to use multiple times

robots.txt

robots.txt is a bot exclusion standard. It was first established in June 1994 to control the access of bots such as web crawler. It is not enforced but if you frequently access banned web pages, it can be understood as a attack.

  • robots.txt location
    • root of the web page
    • e.g. www.naver.com/robots.txt
  • User-agent
    • rules assigned to robots
  • Disallow
    • access banned web page
  • Allow
    • access allowed web pages
  • Crawling may be banned, but there is an exception for the main page
User-agent: *
Disallow: /
Allow: /$
  • Allow access to all bots
    • User-agent: *
    • Allow: /
  • Ban access to all bots
    • user-agent: *
    • Disallow: /
  • Ban access to a specific directory
    • User-agent: *
    • Disallow: /directory_name1/
  • Ban access to a specific file
    • User-agent: *
    • Disallow: /directory_name1/file_name.html
  • Allow access to a specific bots
    • User-agent: specific_bot_name
    • Allow: /directory_name1/directory_name2/
  • Ban access to a specific bot
    • User-agent: specific_bot_name
    • Disallow: /directory_name1/directory_name2/
