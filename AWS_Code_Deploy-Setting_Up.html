AWS Code Deploy - Setting Up

AWS Code Deploy

AWS Code Deploy is a CICD pipeline tool that you can use to create a pipeline to AWS EC2 or other AWS computing instances.

Why choose AWS Code Deploy when there are other choices for CICD pipeline?

A pipeline that I used to use was by using Docker, GitHub Actions, and Watchtower. The problem with this is that it uses docker which could consume a lot of memory, leading to failure of a server. However, by using AWS Code Deploy, you can construct a pipeline so that it runs on the operating system such as Ubuntu right on it.

Step 6️⃣

Install and download the following to EC2.

# Download AWS Corretto
$ sudo wget https://corretto.aws/downloads/latest/amazon-corretto-11-x64-linux-jdk.rpm -O java11.rpm

# Install Java 11 JDK
$ sudo apt install openjdk-11-jdk

# Verify Java version
$ java --version

# Install Ruby
$ sudo apt install ruby-full

# Download Seoul region resource kit file for AWS Code Deploy
$ wget https://aws-codedeploy-ap-northeast-2.s3.ap-northeast-2.amazonaws.com/latest/install

# Grant execute
$ chmod +x ./install

# Install agent
$ sudo ./install auto

# Verify agent status
$ sudo service codedeploy-agent status

Step 7️⃣

Add repository secrets to your GitHub repository.

secrets.PROJECT_NAME: Name of your project

secrets.CICD_BUCKET: Name of your cicd bucket

secrets.CODE_DEPLOY_APP_NAME: Name of your CodeDeploy application

secrets.DEPLOYMENT_GROUP_NAME: name of your CodeDeploy deployment group

Step 8️⃣

Create .yaml for GitHub Actions flow to your repository .github/workflows/.

name: CICD using AWS CodeDeploy

on:
  push:
    branches: [ "master" ]

env:
  PROJECT_NAME: $
  BUCKET_NAME: $
  CODE_DEPLOY_APP_NAME: $
  DEPLOYMENT_GROUP_NAME: $

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v3

      - name: Setup Java JDK 11
        uses: actions/setup-java@v3
        with:
          distribution: 'adopt'
          java-version: '11'

      - name: Grant execute permission for gradlew and deploy.sh
        run: chmod +x gradlew scripts/deploy.sh
        shell: bash

      - name: Build with Gradle
        run: ./gradlew build
        shell: bash

      - name: Make ZIP File
        run: zip -qq -r ./$GITHUB_SHA.zip .
        shell: bash

      - name: Configure AWS credentials
        uses: aws-actions/configure-aws-credentials@v1
        with:
          aws-access-key-id: $
          aws-secret-access-key: $
          aws-region: $

      - name: Upload to S3
        run: aws s3 cp --region ap-northeast-2 ./$GITHUB_SHA.zip s3://$BUCKET_NAME/$PROJECT_NAME/$GITHUB_SHA.zip

      - name: Code Deploy
        run: aws deploy create-deployment --application-name $CODE_DEPLOY_APP_NAME --deployment-config-name CodeDeployDefault.OneAtATime --deployment-group-name $DEPLOYMENT_GROUP_NAME --s3-location bucket=$BUCKET_NAME,bundleType=zip,key=$PROJECT_NAME/$GITHUB_SHA.zip

Step 9️⃣

Create appspec.yml to your repository’s root path.

version: 0.0
os: linux

files:
  - source: /
    destination: /home/ubuntu/project

permissions:
  - object: /home/ubuntu/project/
    owner: ubuntu
    group: ubuntu

hooks:
  AfterInstall:
    - location: scripts/deploy.sh
      timeout: 60
      runas: ubuntu

Step 1️⃣0️⃣

Create deploy.sh to this path scripts/.

#!/usr/bin/env bash

REPOSITORY=/home/ubuntu/project
APPLICATION_YML=/home/ubuntu/application.yml
cd $REPOSITORY

APP_NAME=project
JAR_NAME=$(ls $REPOSITORY/build/libs/ | grep 'app.jar' | tail -n 1)
JAR_PATH=$REPOSITORY/build/libs/$JAR_NAME

CURRENT_PID=$(pgrep -f $APP_NAME)

if [ -z $CURRENT_PID ]
then
  echo "> $APP_NAME process does not exist"
else
  echo "> kill -9 $CURRENT_PID"
  kill -15 $CURRENT_PID
  sleep 5
fi

echo "> $JAR_PATH Deployed"
nohup java -jar $JAR_PATH --spring.conf.location=$APPLICATION_YML --server.port=8080 > /dev/null 2> /dev/null < /dev/null &

Caution ❗️

Make sure you have your application.yml or application.properties in EC2 if you have any environment variables.

Step 1️⃣1️⃣

Finally, push to GitHub!
